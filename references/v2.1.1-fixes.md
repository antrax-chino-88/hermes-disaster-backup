### 问题 1：enricher 未集成到主脚本（硬伤）

**现象**：cron 自动备份的 JSONL 中 726 条 Telegram 消息 chat_id 全空（enrich: 0/726），旧格式手动备份才有 chat_id。

**根因**：`disaster_backup.py` 导出 JSONL 后直接验证，没有调用 `telegram_chat_enricher.py`。

**修复**：在 `_backup_profile()` 导出 JSONL 后、返回前，插入 subprocess 调用 enricher：

```python
enriched_path = BACKUP_DIR / f"messages_{profile}_{ts}_enriched.jsonl"
enricher_script = Path(__file__).parent / "telegram_chat_enricher.py"
if not enricher_script.exists():
    enricher_script = Path(r"C:\Users\admin\AppData\Local\hermes\profiles\wangna\scripts\telegram_chat_enricher.py")
if enricher_script.exists():
    result = subprocess.run(
        [sys.executable, str(enricher_script), str(jsonl_path), str(enriched_path)],
        capture_output=True, text=True
    )
    if result.returncode == 0 and enriched_path.exists():
        shutil.move(str(enriched_path), str(jsonl_path))
```

**关键**：`enriched_path` 不能与原始 `jsonl_path` 重名，否则读写同一文件导致覆盖失败。已作为既有 pitfall 记录在 SKILL.md。

### 问题 3：旧格式残留未清理

**现象**：`messages_20260614_04*.jsonl` 等无 profile 前缀的旧格式文件仍在 D 盘。

**根因**：清理规则 `messages_{profile}_*.jsonl` 不匹配旧格式 `messages_YYYYMMDD_HHMMSS.jsonl`。

**修复**：在 `_backup()` 清理段增加旧格式匹配：

```python
old_fmt_files = sorted(
    [p for p in BACKUP_DIR.glob("messages_2*.jsonl")
     if not any(p.name.startswith(f"messages_{pr}_") for pr, _ in profiles)],
    key=lambda p: p.stat().st_mtime, reverse=True
)
for old in old_fmt_files[10:]:
    old.unlink()
```

### 附赠修复：WAL 模式下 shutil.copy2 漏数据

**现象**：wangna DB count mismatch: 5683 vs 5695（备份后验证 DB 消息数比 JSONL 少 12 条）。

**根因**：Hermes state.db 使用 WAL 模式（Write-Ahead Logging），`.db` 文件本身可能不包含最新提交的数据，未提交的 WAL 记录在 `-wal` 文件中。`shutil.copy2` 只复制 `.db` 文件，漏掉 WAL 数据。

**修复**：用 `sqlite3.backup()` API 替代 `shutil.copy2`，该 API 会自动合并 WAL 数据到目标数据库：

```python
src_conn = sqlite3.connect(str(db_path))
dst_conn = sqlite3.connect(str(db_backup))
with dst_conn:
    src_conn.backup(dst_conn)
dst_conn.close()
src_conn.close()
```

**注意**：必须在原连接 `conn.close()` 之后重新 `sqlite3.connect()` 打开备份连接，因为 sqlite3 backup API 要求两个独立的连接对象。

### 教训

1. **Enricher 是备份流程的一部分，不是独立的手动步骤**。如果 enrich 需要手动跑，cron 自动备份就等于白板。
2. **备份脚本和恢复脚本的表结构必须同步**。新增列时要两边一起改。
3. **WAL 模式下永远不要直接用文件系统拷贝 SQLite 数据库**。这是 SQLite 的基本常识，但在 Windows + Hermes 环境下容易被忽略。
4. **清理规则要覆盖所有历史格式**。升级文件名格式时必须同时处理旧格式残留。
